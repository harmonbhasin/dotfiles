# Project Scaffolds

Reusable files for starting or standardizing projects.

## Profiles

- `base`: general project defaults that apply to most repos.
- `typescript`: TypeScript-specific files. Create this directory when the first TypeScript-only scaffold is ready.
- `python`: Python tooling (ruff, mypy, pytest, pre-commit) for uv-managed projects.

Language profiles layer on top of `base`. For example, `sp python` copies `base` first, then `python`.

When a profile ships a `.gitignore`, `sp` appends it to any existing `.gitignore` (with a `# --- <profile> ---` divider) rather than overwriting — so base's entries and the language profile's entries coexist.

## Usage

Run `sp` from a project root:

```bash
sp
```

That copies `scaffolds/base` into the current directory.

Other forms:

```bash
sp base
sp typescript
sp python
sp --list
sp --force typescript
```

By default, `sp` refuses to overwrite existing files. Use `--force` only when replacing project files is intentional.

After copying, `sp` also symlinks `CLAUDE.local.md -> AGENTS.override.md` when the override file is present and no `CLAUDE.local.md` already exists. Both paths are gitignored, so this gives Claude Code a local agent config without committing it.

## Python profile

`sp python` ships a `pyproject.toml.fragment` rather than a full `pyproject.toml`, so it composes cleanly with `uv init`:

```bash
uv init
sp python
# Then merge pyproject.toml.fragment into pyproject.toml and delete the fragment.
uv sync
uv run pre-commit install
```

The fragment contains only the `[dependency-groups]`, `[tool.ruff]`, `[tool.mypy]`, and `[tool.pytest.ini_options]` tables — the `[project]` and `[build-system]` tables generated by `uv init` are left alone.

## Shell Setup

The command lives in `scaffolds/sp.sh` and is sourced by both `bash/.bashrc` and `zsh/.zshrc`.
